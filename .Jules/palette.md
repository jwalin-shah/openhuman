## 2024-04-27 - Missing ARIA labels on Icon-only Buttons Pattern
**Learning:** Found a systemic pattern across modals, toast notifications, and actionable cards where close/dismiss/snooze/complete icon-only buttons lacked `aria-label`s. Because these buttons contained only SVG icons, screen readers would not announce any discernible text, making critical interactions (like dismissing a toast or closing a modal) inaccessible.
**Action:** Created an automated script to scan for `<button>` tags without visible text and `aria-label`s, which successfully surfaced 7 missing labels. Added appropriate `aria-label` attributes to these components. Moving forward, always check icon-only buttons (often stylized with SVG children) for `aria-label` attributes as a default accessibility check when adding or modifying UX components.
